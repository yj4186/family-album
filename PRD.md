# PRD: 가족앨범 (Family Album)

- 작성일: 2026-07-07
- 상태: MVP
- 참고 문서: [initial.md](./initial.md)

## 1. 개요

**제이(나)**와 **꿈이(강아지)**로 구성된 가족을 소개하는 개인용 가족 웹앨범이다. 회원가입/로그인 없이 누구나 접속해 가족을 소개받고 사진을 볼 수 있는 정적 웹사이트를 MVP로 제작한다.

## 2. 목표 / 목표가 아닌 것

### 2.1 MVP 범위 (In-scope)

- 홈 / 소개 / 갤러리 3개 페이지
- 상단 좌측 SVG 로고, 하단 더미 데이터 푸터
- 반응형 레이아웃 (모바일 / 데스크톱)
- 정적 데이터 기반 (백엔드, DB 없음)

### 2.2 범위 제외 (Out-of-scope, 차후 고려)

- 로그인/회원 관리
- 사진 업로드, 관리자 CMS
- 댓글/방명록
- 실시간 데이터베이스 연동
- 다국어 지원

## 3. 타겟 사용자

- 가족(제이) 본인 및 지인들에게 공유하는 개인 웹사이트
- 별도 사용자 세그먼트/권한 구분 없음 (단일 퍼블릭 뷰)

## 4. 정보 구조 (IA)

```
/           홈
/about      소개
/gallery    갤러리
```

### 4.1 공통 레이아웃

- **헤더**: 좌측 SVG 로고(클릭 시 홈으로 이동) + 우측 내비게이션(홈/소개/갤러리)
- **본문**: 페이지별 콘텐츠
- **푸터**: 더미 데이터 (4.4 참고)

### 4.2 홈 (`/`)

- 히어로 영역: 가족 대표 이미지(강아지 사진) + 환영 문구 (예: "제이와 꿈이의 가족앨범에 오신 것을 환영합니다")
- 소개 페이지 / 갤러리 페이지로 이동하는 CTA 버튼 2개

### 4.3 소개 (`/about`)

- 가족 소개 인트로 문단 (짧은 텍스트)
- 구성원별 프로필 카드 2개 (제이, 꿈이)
  - 프로필 사진, 이름, 역할(예: "가족", "반려견"), 한 줄 소개
- 꿈이의 프로필 사진도 Pexels URL 사용

### 4.4 갤러리 (`/gallery`)

- Pexels에서 가져온 강아지 관련 사진 URL을 사용한 그리드형 갤러리 (최소 9~12장)
- 각 이미지에는 alt 텍스트 필수
- 이미지 클릭/호버 시 확대 효과 (CSS 기반 hover scale). 별도 라이트박스/모달은 MVP 범위 제외
- 사진 출처: Pexels 직링크 URL (저작자 표시 불필요, 무료 사용 가능 라이선스)

### 4.5 푸터 (모든 페이지 공통)

더미 데이터로 구성:

- 저작권 문구: `© 2026 가족앨범. All rights reserved.`
- 더미 SNS 링크 (Instagram, YouTube 등 아이콘, 실제 연결 없이 `#` 처리)
- 더미 연락처: `contact@familyalbum.example.com`

## 5. 로고 사양

- 형식: SVG (인라인 컴포넌트로 구현)
- 디자인: 푸른색 계열(`#4A90D9` ~ `#2C5F9E` 톤) 구름 모양 아이콘 + "가족앨범" 워드마크
- 위치: 헤더 좌측, 클릭 시 홈(`/`)으로 이동

## 6. 데이터 모델 (정적)

백엔드 없이 프론트엔드 내 정적 JSON/JS 상수로 관리한다.

**`family.json`** — 소개 페이지용

```json
[
  { "id": "jay", "name": "제이", "role": "가족", "photoUrl": "<Pexels URL>", "bio": "한 줄 소개" },
  { "id": "kkumi", "name": "꿈이", "role": "반려견", "photoUrl": "<Pexels URL>", "bio": "한 줄 소개" }
]
```

**`gallery.json`** — 갤러리 페이지용

```json
[
  { "id": 1, "url": "<Pexels 강아지 사진 URL>", "alt": "설명 텍스트" }
]
```

## 7. 기술 스택

- **프레임워크**: React + Vite
- **라우팅**: React Router
- **스타일링**: CSS Modules (또는 순수 CSS)
- **이미지**: Pexels 직링크 URL (별도 이미지 서버/스토리지 없음)
- **배포 가정**: 정적 호스팅 (예: GitHub Pages, Vercel, Netlify 등)

## 8. 비기능 요구사항

- 반응형: 모바일(360px~)/태블릿/데스크톱 대응
- 접근성: 모든 이미지에 alt 텍스트 제공
- 브라우저 지원: 최신 Chrome/Edge/Safari 기준
- 성능: 별도 최적화(이미지 lazy loading 등)는 MVP 이후 고려 가능

## 9. 향후 확장 아이디어 (Out-of-scope, 참고용)

- 사진 업로드 및 앨범/카테고리 분류 기능
- 방명록/댓글 기능
- 관리자 페이지를 통한 콘텐츠 관리(CMS)
- 실제 SNS 연동 및 실제 연락처 정보로 교체
- 사진 라이트박스/슬라이드쇼 뷰어

## 10. 성공 기준 (MVP)

- 3개 페이지가 정상적으로 라우팅되고 반응형으로 표시됨
- 로고(SVG)와 푸터(더미 데이터)가 모든 페이지에 공통 표시됨
- 갤러리에 Pexels 강아지 사진이 최소 9장 이상 정상 노출됨
- 소개 페이지에 제이/꿈이 프로필이 사진과 함께 표시됨
